깃 기초 명령어
git init
-> 빈 폴더 하나를 깃 저장소로 만드는 명렁어다.

저장소 상태 확인
git status
-> 현재 변경사항, 스테이징된 파일 등 
   저장소 정보 출력

변경사항 스테이지에 추가
git add 
<file_path>
-> 해당 파일을 스테이징 한다.

git add .
-> 모든 변경사항을 스테이징한다.

스테이징한 내역을 확정하여 버전 생성
git commit -m "남기고 싶은 메시지"

============== 2단계 =============================

Github와 로컬의 리포지토리를 연결하여, 로컬의 히스토리를 백업해보자.

로그인, 리포 생성

원격 저장소의 링크를 받아서
git remote add <원격저장소별칭>
<원격저장소링크>

원격 저장소와의 연결 정보 확인
git remote -v

원격 저장소로 로컬 저장소의 최신 변경 사항을 적용(동기화)
git push <원격별칭> <브랜치명>

사용자 설정 추가
git config
	--global
	-> 모든 위치에서 이 별칭 적용

	--local
	-> 현재 폴더 위치에 별칭 적용
	user.name "유저명"

	user.email "이메일정보"
